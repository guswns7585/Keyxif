================================================================================
 Keyxif 웹앱 — PC(데스크톱) UX/UI 개선 내역
 작성일: 2026-07-13
================================================================================

[ 요약 ]
안드로이드 앱을 1:1 이식한 초기 버전은 모바일 세로 화면 기준(폭 760px 고정
단일 컬럼)이라, 넓은 PC 화면에서는 양옆이 비고 편집 결과를 저장 단계까지
확인할 수 없었습니다. 아래 개선으로 PC에서 "보면서 편집하는" 경험으로 바꿨고,
모바일 레이아웃은 그대로 유지했습니다(전부 화면 폭 900px 이상에서만 적용되는
반응형 방식).


--------------------------------------------------------------------------------
 1. 2단 레이아웃 + 실시간 미리보기 패널
--------------------------------------------------------------------------------
- 변경: 정보(BuildInfo)·템플릿(Template) 단계에서 화면을 좌/우 2단으로 분할.
        좌측은 편집 폼/템플릿 목록, 우측은 현재 사진에 선택한 템플릿을 적용한
        결과 카드를 실시간으로 렌더링해 표시(파일명·대표 색상 팔레트 포함).
- 효과: 저장 단계까지 가지 않아도 편집·템플릿 선택 즉시 결과 확인.
- 세부: 폭 900px 이상에서만 2단, 그 미만(모바일)은 기존 단일 컬럼 유지.
- 파일: main.js(패널 주입 로직), ui-main.js(buildPreviewPane), app.css(.workarea/.preview-pane)


--------------------------------------------------------------------------------
 2. 화면 폭 확장 + 좌우 마진 축소
--------------------------------------------------------------------------------
- 변경: 앱 최대 폭 760px → 1440px. 미리보기 패널 폭을 최대 680px까지 확대
        (clamp(440px, 44vw, 680px) — 화면 폭에 맞춰 반응형).
        미리보기 이미지 최대 높이도 상향.
- 효과: 넓은 화면을 충분히 활용, 미리보기가 크고 시원하게 보임.
- 사진/저장/갤러리/설정 등 패널이 없는 화면은 가독성을 위해 중앙 정렬 폭 제한.
- 파일: app.css (데스크톱 미디어쿼리)


--------------------------------------------------------------------------------
 3. 스크롤해도 미리보기 고정 (따라오기)
--------------------------------------------------------------------------------
- 문제: 데스크톱에서 전체 페이지가 스크롤돼, 폼을 내리면 미리보기까지 함께
        위로 밀려 사라짐.
- 해결: 앱 높이를 화면 높이에 고정(height:100dvh)해 "좌측 폼만 내부 스크롤"
        되도록 변경. 상단바·하단 버튼바·우측 미리보기는 항상 제자리 고정.
        미리보기 영역이 패널의 남은 높이를 꽉 채우도록 조정(스크롤 없이 전체 노출).
- 파일: app.css (#app height, .preview-pane-stage)


--------------------------------------------------------------------------------
 4. 스크롤 위치 보존 (맨 위로 튐 현상 수정)
--------------------------------------------------------------------------------
- 문제: 정보를 채울 때 값(칩/보강판/마운트 등)을 하나 선택할 때마다 화면이
        맨 위로 튐. (값 선택 → 전체 화면 재렌더 → 스크롤 위치 초기화)
- 해결: 재렌더 직전 스크롤 위치를 저장하고 재렌더 후 복원.
- 검증: 폼을 480px 내린 상태에서 칩을 눌러도 480px 그대로 유지됨 확인.
- 파일: main.js (captureScroll / restoreScroll)


--------------------------------------------------------------------------------
 5. 검색창 PC 가시성 개선
--------------------------------------------------------------------------------
- 변경: 하우징/스위치/키캡 입력칸의 "검색" 버튼을 흐린 텍스트 버튼 →
        돋보기 아이콘 + 컬러(tonal) 버튼으로 강조해 눈에 잘 띄게 함.
- 데스크톱에서는 검색창을 하단 시트 대신 화면 중앙 모달(폭 560px)로 띄워
        크고 명확하게 표시(모바일은 기존 하단 시트 유지).
- 파일: ui-main.js(검색 버튼 마크업/아이콘), app.css(.field-search-btn, 데스크톱 .sheet)


--------------------------------------------------------------------------------
 6. 검색 결과 레이아웃 일관화 (커졌다 작아졌다 방지)
--------------------------------------------------------------------------------
- 문제: 검색 결과 개수에 따라 검색창(시트/모달)이 커졌다 작아졌다 함.
- 해결: 검색창을 고정 높이로(데스크톱 600px / 모바일 74vh), 결과 목록은 남은
        공간을 채우고 내부만 스크롤. 결과가 1개든 100개든 창 크기 동일.
- 검증: 결과 80개일 때와 3개일 때 모달 높이 모두 600px로 동일 확인.
- 파일: app.css(.sheet 고정 높이, .sheet-list flex), ui-main.js(인라인 높이 제거)


--------------------------------------------------------------------------------
 7. 앱 이름 클릭 → 메인 화면 복귀
--------------------------------------------------------------------------------
- 변경: 좌측 상단 "Keyxif" 로고를 클릭하면 첫 화면(사진 단계)으로 이동.
- 파일: main.js (topbar-title 버튼화)


--------------------------------------------------------------------------------
 8. 알람(스낵바) 위치 → 우하단
--------------------------------------------------------------------------------
- 변경: 하단 중앙에서 뜨던 알림을 우측 하단 코너에서 슬라이드 인 되도록 변경
        (데스크톱 우측 24px, 모바일 16px, 하단 버튼바와 겹치지 않게 유지).
- 파일: app.css (#snackbar-host)


--------------------------------------------------------------------------------
 9. 마우스 친화 요소 (부가)
--------------------------------------------------------------------------------
- 템플릿/갤러리/저장/설정 카드, 칩, 목록 행에 마우스 호버 반응 추가.
- 템플릿·저장·갤러리 그리드는 넓은 화면에서 열 수 자동 증가.
- 파일: app.css (데스크톱 미디어쿼리 hover 규칙)


================================================================================
 참고 / 운영
================================================================================
- 개선은 모두 "폭 900px 이상"에서만 적용되는 반응형이라 스마트폰 화면은
  기존 그대로 동작합니다.
- 파일 구조:
    index.html          진입점 (모듈 스크립트 로드, 캐시버전 ?v= 관리)
    app.css             테마 + 레이아웃 (데스크톱 미디어쿼리 포함)
    main.js             앱 셸(상단바/단계바/라우팅/스크롤·포커스 보존)
    ui-main.js          4개 메인 화면 + 미리보기 패널
    ui-gallery.js       완성 이미지 갤러리
    ui-settings.js      설정
    store.js            상태 관리
    search.js/palette.js/db.js/data/*  검색·색상분석·저장소·데이터
    dist/Keyxif.html    위 전체를 하나로 합친 단일 파일(배포용/오프라인용)
